## 神经网络是什么

神经网络听起来像一个很复杂的东西，但在刚开始学习时，可以先把它理解成一种特殊的“预测函数”。

所谓预测函数，就是给它一些输入，它会给出一个输出。比如我们想预测一次小测能不能通过，可以把“学习时长”和“复习次数”交给模型，让模型给出一个判断：

```text
学习时长、复习次数 -> 模型 -> 是否通过小测
```

这里的模型如果是神经网络，就意味着：它不是靠我们一条条写死规则来判断，而是可以通过训练，逐渐调整自己内部的一些数字，让预测结果越来越接近真实答案。

所以，本文先给出一个朴素定义：

```text
神经网络是一种可以通过训练调整参数的预测函数。
```

这个定义里有三个关键词：预测、训练、参数。预测是它要完成的任务；训练是让它变好的过程；参数是它内部会被训练调整的数字。后面的文章会逐步展开训练闭环，这一篇只先把“神经网络到底是什么”讲清楚。

## 从一个预测问题开始

假设我们关心一个很小的学习场景：一个同学今天学习了 2 小时，复习了 1 次，我们想预测他明天的小测是否能通过。

可以把输入写成：

```text
学习时长 = 2 小时
复习次数 = 1 次
```

我们希望得到的输出是：

```text
小测是否通过
```

这就是一个预测问题。它不要求模型“理解人生”，也不要求模型像老师一样分析所有细节。它只需要根据给定的信息，给出一个可能的判断。

当然，只看两个数字就判断小测结果，肯定会有不确定性。有人学习效率高，2 小时就够；有人虽然复习了 1 次，但复习得很粗略；也可能题目当天特别难。真实世界里，预测很少是百分之百确定的。模型做的是：在已有信息下，给出一个尽量合理的结果。

为了让问题更具体，我们可以把预测结果先简化成两种：

```text
通过
不通过
```

神经网络要学的，就是从类似这样的样本中找规律：

```text
学习时长 = 2 小时，复习次数 = 1 次 -> 通过还是不通过？
```

一开始，它可能判断得很随意；训练之后，如果样本足够合适，它就会逐渐学到：学习时长、复习次数和小测结果之间大概有什么关系。

## 模型不是手写规则

面对这个问题，我们也可以不用神经网络，直接手写规则。例如：

```text
如果学习时长 >= 2 小时，并且复习次数 >= 1 次，就预测通过。
否则预测不通过。
```

这条规则很直观，也容易解释。但它有一个明显问题：规则是人提前写死的。也就是说，真正决定结果的不是数据，而是我们自己的假设。

如果后来发现很多同学学习 1.5 小时也能通过，这条规则就太严格了；如果发现只复习 1 次远远不够，这条规则又太宽松了。我们可以继续改规则，比如加上更多条件：

```text
如果学习时长 >= 2.5 小时，预测通过。
如果学习时长在 1 到 2.5 小时之间，还要看复习次数。
如果复习次数太少，预测不通过。
```

规则越写越多，问题也会越来越麻烦。每一条边界该放在哪里，都要人来决定。学习时长是 1.9 小时和 2.0 小时，真的会突然从“不通过”变成“通过”吗？复习 1 次和 2 次之间，又该差多少？

训练模型的思路不同。我们不直接告诉模型完整规则，而是给它看许多例子：

```text
学习时长 = 1 小时，复习次数 = 0 次 -> 不通过
学习时长 = 2 小时，复习次数 = 1 次 -> 通过
学习时长 = 3 小时，复习次数 = 1 次 -> 通过
学习时长 = 1.5 小时，复习次数 = 2 次 -> 通过
```

模型根据这些样本，自己调整内部数字，让它对训练样本的预测尽量接近真实结果。人仍然要设计模型、准备数据、选择训练方式，但“具体怎样从输入走到预测”这件事，不再完全由一条条手写规则决定。

## 参数：模型里会被训练调整的数字

那模型内部到底在调整什么？答案是：参数。

参数可以先理解成模型里面的一些数字。这些数字不会直接出现在输入里，也不是我们每次预测时临时填写的内容。它们更像模型内部的旋钮，决定了模型如何看待不同输入。

还是用小测预测的例子。模型可能需要决定：

```text
学习时长的重要性有多大？
复习次数的重要性有多大？
整体上，预测通过应该更容易还是更谨慎？
```

这些“重要性”和“倾向”最终都可以由一些数字来控制。训练开始前，这些数字通常还不合适，所以模型的预测可能很差。训练过程中，模型会一次次比较自己的预测和真实答案，然后调整这些数字。

比如，对于“学习时长 = 2 小时，复习次数 = 1 次”的样本，如果真实结果是通过，而模型预测成不通过，那么训练过程就会推动内部参数往某个方向改变，让下次遇到类似情况时，模型更可能预测通过。

这里不需要知道参数具体怎么改，也不需要推公式。现在只要抓住一个直觉：

```text
参数不是输入，而是模型内部会被训练调整的数字。
```

输入是“学习时长 = 2 小时、复习次数 = 1 次”；参数是模型内部用来处理这些输入的数字。输入每个样本都可能不同，参数则是在训练中逐渐形成，并在训练完成后保留下来，用于之后的新预测。

## 神经网络可以看成一个函数

如果把神经网络看成函数，它的大致形式可以写成：

```text
预测结果 = 模型(学习时长, 复习次数)
```

当我们把“学习时长 = 2 小时，复习次数 = 1 次”放进去，模型会输出一个预测。这个预测可以是“通过 / 不通过”，也可以先是一个分数，再根据分数判断更像哪一类。

更完整一点，可以这样看：

```text
输入
-> 使用参数进行计算
-> 得到预测
```

或者放回小测例子：

```text
学习时长 = 2 小时，复习次数 = 1 次
-> 神经网络
-> 预测小测是否通过
```

这和普通函数相似：输入进去，输出出来。但神经网络和普通手写函数的关键区别在于，普通函数的逻辑通常由人直接写好，而神经网络函数内部有许多参数，这些参数可以通过训练从数据中调整出来。

可以把它想成一个开始并不聪明的函数。它一开始只是能计算，但算得不一定准。训练给了它一次次修正自己的机会：预测错了，就调整参数；再预测，再比较；再调整。经过很多轮之后，它处理输入的方式会发生变化，预测也可能变得更可靠。

所以，“神经网络是函数”这句话不是说它很简单，而是说我们可以先用函数的视角理解它：它接收输入，经过内部计算，产生输出。至于它为什么能学、怎么学、训练闭环如何让参数变好，后面再继续拆开。

## 小结

这一篇只建立最基础的直觉：神经网络是一种可以通过训练调整参数的预测函数。

在“小测是否通过”的例子里，输入可以是：

```text
学习时长 = 2 小时
复习次数 = 1 次
```

输出是对小测结果的预测：

```text
通过 / 不通过
```

如果我们手写规则，就要提前决定“学习多久算够”“复习几次算够”。这种方式清楚直接，但规则边界完全依赖人的设定。

训练模型则是另一种思路：我们给模型看样本，让它根据预测和真实答案之间的差距，调整内部参数。参数就是模型内部会被训练调整的数字，它们决定模型如何把输入变成预测。

可以把主线记成：

```text
输入 -> 模型 -> 预测
```

也可以更具体地记成：

```text
学习时长、复习次数 -> 神经网络 -> 小测是否通过
```

理解到这里，就已经抓住了神经网络的第一层含义：它不是一堆手写判断规则，而是一个带有可调整参数的预测函数。接下来再看训练时发生了什么，就会自然得多。
