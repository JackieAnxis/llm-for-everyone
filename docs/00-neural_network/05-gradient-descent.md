## 梯度下降是什么

上一章说到，损失函数会把模型的预测和真实答案之间的差距，变成一个叫“损失”的数字。损失越小，通常表示预测越接近真实答案；损失越大，表示预测越差。

但训练并不会停在“知道错了”这一步。模型还需要继续回答一个问题：既然这次预测不够好，内部的权重和偏置应该怎样改，下一次才可能更好？

梯度下降就是用来做这件事的一种方法。它的目标很直接：不断调整参数，让损失尽量变小。

可以把损失想象成一座山上的高度。参数的位置不同，损失的高度也不同。训练的目标不是爬到更高的地方，而是往更低的地方走。梯度提供的是方向信息，帮助我们判断参数往哪边改，可能会让损失变小。

这一章只做直觉解释，不推导梯度公式。初学阶段先记住一件事就够了：梯度不是最终答案，而是提醒模型“往哪边调整更有希望”的线索。

## 参数不是一次调对的

神经网络里有很多参数。前面我们提到过，权重会影响输入信息的重要程度，偏置会影响神经元整体更容易被激活还是更不容易被激活。模型的预测，就是这些参数共同作用之后的结果。

刚开始训练时，参数通常还不合适。它们可能让模型过于乐观，也可能让模型过于保守。比如在小测预测中，一个学生学习时间不少，复习次数也可以，但模型却只预测他通过的可能性是 `0.2`。如果真实答案是通过，也就是标签为 `1`，这个预测就明显太低了。

这时我们会希望模型下次看到类似情况时，预测值能更高一些。直觉上，和“学习时长”“复习次数”相关的某些权重，可能需要被调大。这样，这些输入对“通过”这个结果的推动作用就会更明显。

不过，真实训练不会只看这一个样本，也不会凭一句“调大一点”就随意修改所有参数。一次调整通常只是一小步，参数是在很多轮训练中慢慢变得更合适的。

## 往损失变小的方向走

有了损失之后，我们知道模型现在错得多严重；有了梯度之后，我们进一步得到方向信息。

这个方向信息可以帮助模型判断：某个参数稍微变大或变小时，损失可能怎样变化。训练时，我们希望选择能让损失变小的方向，而不是让损失越来越大。

继续看小测例子。假设真实标签是 `1`，模型预测通过概率只有 `0.2`。这个预测太低，说明模型对“会通过”的判断不够强。梯度会帮助训练过程判断，哪些权重应该往上调，哪些权重应该往下调，哪些参数暂时只需要很小的变化。

这不是说所有权重都要一起变大。某些权重调大可能有帮助，某些权重调大反而可能让别的样本预测变差。梯度下降做的事情，就是借助损失给出的反馈，沿着整体上更可能让损失变小的方向更新参数。

## 学习率：每次走多大一步

知道方向还不够，还要决定每次走多大一步。这个“步子大小”由学习率控制。

学习率是训练中的一个重要设置，它决定每次更新参数时，根据梯度判断出的更新方向移动多少。学习率大，参数每次变化就大；学习率小，参数每次变化就小。

如果学习率太大，模型可能一下子跨过了比较好的位置，损失反而忽高忽低。如果学习率太小，模型每次只挪动一点点，训练会很慢。

所以，学习率控制的是更新幅度。梯度告诉模型大致往哪边走，学习率告诉模型这一步走多远。两者配合起来，才形成一次具体的参数更新。

初学阶段先记住这个直觉：方向选得对，步子太大或太小也都会影响训练效果。

## 梯度下降在训练闭环里的位置

现在可以把前几章连起来看了。

训练通常不是单独的一步，而是一个反复进行的闭环：

```text
输入样本
-> 前向传播得到预测
-> 损失函数计算损失
-> 根据梯度判断参数调整方向
-> 按学习率更新参数
-> 再次进入下一轮
```

前向传播负责回答“模型现在会预测什么”。损失函数负责回答“这次预测错得多严重”。梯度下降负责回答“参数接下来应该怎样改，损失才更可能变小”。

这个闭环会重复很多次。训练过程不保证每一步都完美，但大的方向是清楚的：通过观察损失，并沿着让损失变小的方向更新参数，让模型逐渐学会更合适的判断方式。

## 小结

损失函数告诉我们模型预测得好不好，而梯度下降进一步告诉我们参数应该怎样调整。梯度提供方向信息，帮助判断参数往哪边改会让损失变小。

在小测预测中，如果真实答案是通过，模型却预测得太低，那么和学习时长、复习次数等输入相关的某些权重，可能需要被调大。这样模型下次看到类似样本时，预测通过的可能性就可能更接近真实标签。

学习率控制每次更新参数的幅度。它决定沿着根据梯度判断出的更新方向走多大一步：太大可能不稳定，太小可能训练很慢。

本章只是建立直觉：训练并不是一次把参数调对，而是在“预测、计算损失、判断方向、更新参数”的闭环中反复前进。梯度下降就是这个闭环里负责让参数朝损失变小方向移动的核心方法。
